# Olex2 Commands and Syntax Reference

This document provides a summary of commands available in Olex2, their syntax, and usage, based on the "Commands.html" documentation.

## Introduction

Commands in Olex2 can be typed directly into the input field, with text and responses appearing in the bottom left corner of the main window. The display scrolls up, and the number of visible lines can be set with `lines n`. `Ctrl+T` toggles between molecule-only, text-only, and combined views. Output can also be viewed in a text editor with the `text` command.

Many Olex2 commands are four-letter codes, similar to SHELX. SHELX commands (ShelXP, ShelXL, ShelXS) are generally interpreted by Olex2 to build its internal structure model. Commands auto-complete with the `TAB` key.

## Understanding the Syntax

*   **Selection**: If atoms are selected on screen, commands acting on a selection apply only to those atoms. Without selection, they apply to all atoms. Supplying a list of atom names is also possible. Successful commands usually clear the selection.
*   **Mode**: When Olex2 is in a specific "Mode" (e.g., `mode grow`), the chosen action applies to subsequently clicked atoms. The mouse pointer changes to indicate an active mode. Press `Esc` to exit a mode.
*   **Syntax Conventions**:
    *   `{a, b, c}`: Choice between a, b, or c. Example: `fix {occu, xyz, Uiso} [atoms]` means `fix occu [atoms]`, `fix xyz [atoms]`, or `fix Uiso [atoms]`.
    *   `[val=2]`: Optional parameter. If not supplied, the default value (e.g., 2) is used.
    *   `-k`: An option switch.
    *   `i`: Italic characters denote variables.
    *   `[atoms]`: Optional list of atoms. Selected atoms are automatically included. If none selected, applies to all. Atom names can be typed.
    *   `atoms`: Compulsory list of atoms. Selected atoms are automatically included, or names can be typed.
    *   `Capital Letters`: Used for commands directly affecting the structure model in refinement (e.g., `SADI`). These appear in the ShelX input file (case-insensitive when typed).
    *   **Example Commands**: Shown as `refine 4 20`, meaning refine for 4 cycles and show 20 Q-peaks.

### Changing the Model View

| Command     | Syntax                                                                 | Description                                                                                                                                                                                                                                                                                                                                                        |
|-------------|------------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **`matr`**  | `[1,2,3 or abc]` or `[abc a1b1c1]` or `[x11 x12 x13 y11 y12 y13 z11 z12 z13]` | Orients model along crystallographic directions (a, b, c, or custom like 123). Can use two directions (from-to) or a full Cartesian matrix. No arguments: prints current matrix. Examples: `matr 1` (along a), `matr 100 011` (along 011-100). Signed/multi-digit directions need same length components (e.g., `120101`, `-1+1+1`). |
| **`rota`**  | `[axis angle]` or `[x y z angle increment]`                            | Rotates view around specified axis (x, y, z) or performs continuous rotation. X=horizontal, Y=vertical, Z=out of screen. Examples: `rota x 90` (rotates 90° around X), `rota 0 0 1 90 1` (rotates in screen plane 90° with 1° increment).                                                                             |
| **`direction`**|                                                                        | Prints current normal in crystallographic coordinates and tries to match it to a crystallographic direction.                                                                                                                                                                                                                                                         |
| **`mpln`**  | `[atoms]` `[-n]` `[-r]`                                                | Finds best plane through selection/given atoms (or all visible if none given). `-n`: sets view along plane normal. `-r`: creates a regular plane.                                                                                                                                                                                                          |

*Mouse Controls for View:*
*   **Rotate**: Left mouse button + move (or Shift + arrow keys).
*   **Rotate around Z**: Ctrl + Left mouse button + move.
*   **Zoom**: Right mouse button (or Shift + Home/End or Alt + Left mouse button).
*   **Shift in View Plane**: Ctrl + Shift + Right mouse button.

### Fixed/Refined Parameters

| Command     | Syntax                  | Description                                                                                                                                                              |
|-------------|-------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **`fix`**   | `{occu, xyz, Uiso} [atoms]` | Fixes specified parameters (occupancy, coordinates, ADP) so they are not refined. Examples: `fix occu 0.5` (sets & fixes occu to 0.5 for selection), `fix xyz` (fixes coords). |
| **`free`**  | `{occu, xyz, Uiso} [atoms]` | Opposite of `fix`. Makes specified parameters refineable.                                                                                                                 |
| **`mode fixu`** |                       | Fixes Uiso or ADP for subsequently clicked atoms.                                                                                                                         |
| **`mode fixxyz`**|                    | Fixes coordinates for subsequently clicked atoms.                                                                                                                         |
| **`mode occu occupancy_to_set`**|       | Sets occupancy to the provided value for subsequently clicked atoms.                                                                                                      |
*Note: `labels -f` shows fixed parameters; `labels -f -r` also shows occupancy of riding atoms.*

### Atom Connectivity Table Manipulation

| Command      | Syntax                             | Description                                                                                                                                                                                                                                                           |
|--------------|------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **`conn`**   | `n [r] atoms`                      | Sets max bonds for specified `atoms` to `n` and default bond radius to `r`. Examples: `conn 5 $C` (max 5 bonds for C), `conn 1.3 $C` (bond radius 1.3 for C), `conn 5 1.3 $C` (both).                                                                                |
| **`compaq`** | `[-a] [-c] [-q]`                   | Moves asymmetric unit fragments close. Default: assembles around largest. `-a`: assembles broken fragments. `-c`: considers atom-atom distances. `-q`: moves Q-peaks close to atoms.                                                                                    |
| **`addbond`**| `A1 A2` or `selected atoms`        | Adds a bond to connectivity for specified atoms (works with symmetry equivalents).                                                                                                                                                                                    |
| **`delbond`**| `A1 A2` or `selected bond(s)`      | Removes selected bonds from connectivity.                                                                                                                                                                                                                             |
| **`sort`**   | `[m] [l] [p] [h] atoms` or `[s] [h] [m] moiety` | Sorts atoms/moieties. Atom sort: `-m` (mass), `-l` (label), `-p` (part), `-h` (H independent). Moiety sort: `-s` (size), `-h` (heaviest atom), `-m` (mol. weight). Example: `sort +m1 F2 F1 moiety +s`. 'moiety' alone splits atoms into moieties without sorting. |
*Note: Olex2 displays altered connectivity if structure is grown/packed.*

### Symmetry Operations

| Command       | Syntax                                     | Description                                                                                                                                                                                                                                                                                       |
|---------------|--------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **`lstsymm`** |                                            | Prints symmetry operations and codes for the current structure.                                                                                                                                                                                                                                   |
| **`envi`**    | `[r=2.7Å] A1` or `one selected atom [-h] [-q]` | Prints atoms within radius `r` around specified atom. `-h`: adds H atoms. `-q`: adds Q-peaks. Uses first atom if multiple selected.                                                                                                                                                                |
| **`mode grow`**| `[-s] [-v] [-b]` (see docs for `-r` also)  | Displays directions for molecule growth. `-s`: shows short interaction directions. `-v [2.0Å]`: shows directions to molecules within `v` of VdW radii. `-r`: shows directions to symm. equiv. atoms within 15Å. | **`mode pack`**|                                            | Displays symmetry equivalent asymmetric units as tetrahedra; clicking generates them.                                                                                                                                                                                                               |
| **`sgen`**    | `symmetry_operation atoms`                 | Generates symmetry equivalents of specified/all atoms using the operation (e.g., `1_555`, `1555`, or `-1+X,Y,Z`). | **`pack`**    | `from to`                                  | Equivalent to `pack from to from to from to`. | **`pack cell`**|                                            | Shows unit cell content. Use with `grow -w` to show all ASU contributing to the unit cell.                                                                                                                                                                                                           |
| **`pack r`**  | `r`                                        | Packs fragments within radius `r` of selected atom(s) or ASU center of gravity.                                                                                                                                                                                                                   |
| **`grow`**    | `[atoms] [-w]`                             | Grows all/given atoms. For polymers, continues until symmetry elements repeat. `-w`: applies symmetry operations used to other ASU fragments (e.g., to generate solvent molecules after main molecule growth).                                                                                      |
*Notes: Deleted atoms after growing can redefine the ASU. `labels -l -i`: labels only original molecule. Right-click bond > Graphics > Select Group(s) selects all bonds/atoms of that type in grown structure.*

### Disorder Modelling: Constraints and Restraints

| Command      | Syntax                                    | Description                                                                                                                                                                                                                                                                                                                                                                                       |
|--------------|-------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **`EXYZ`**   | `atom_types [-EADP] [-lo]` (for selected atom) | Makes selected site shared by atoms of specified types. `-EADP`: adds EADP for shared atoms. `-lo`: links occupancy via free variable.                                                                                                                                                                                                                                                              |
| **`EADP`**   | `atoms`                                   | Makes ADPs of specified atoms equivalent.                                                                                                                                                                                                                                                                                                                                                         |
| **`SADI`**   | `atoms` or `bonds [esd=0.02]`             | Makes distances of selected bonds/atom pairs similar within `esd`. One selected atom: assumes regular molecule (e.g., PF6), adds similarity restraints for P-F, F-F. Three selected atoms (A1,A2,A3): similarity for A1-A2 and A2-A3.                                                                                                                                                             |
| **`DFIX`**   | `d atom_pairs` or `pairwise_selection [esd=0.02]` | Fixes length of selected bonds/atom pairs to `d` with `esd`. One selected atom: fixes all outgoing bonds. Three selected atoms (A1,A2,A3): fixes A1-A2 and A2-A3.                                                                                                                                                                                                                             |
| **`DANG`**   | `d atom_pairs` or `pairwise_selection [esd=0.04]` | Similar to DFIX, distance restraints for selected bonds/atom pairs.                                                                                                                                                                                                                                                                                                                             |
| **`tria`**   | `d1 d2 angle [esd=0.02]`                  | For bond pairs sharing an atom or atom triplets, generates two DFIX and one DANG. Example: `tria 1 1 180 C1 C2 C3` generates `DFIX 1 0.02 C1 C2 C2 C3` and `DANG 2 0.04 C1 C3` (calculates DANG distance from d1, d2, angle).                                                                                                                                                                   |
| **`FLAT`**   | `[atoms] [esd=0.1]`                       | Restrains given fragment to be flat within `esd`.                                                                                                                                                                                                                                                                                                                                                 |
| **`CHIV`**   | `[atoms] [val=0] [esd=0.1]`               | Restrains chiral volume of provided group to `val` within `esd`.                                                                                                                                                                                                                                                                                                                                  |
| **`SIMU`**   | `[d=1.7] [esd12=0.04] [esd13=0.08]`       | Restrains ADPs of all 1,2 and 1,3 pairs within given atoms to be similar.                                                                                                                                                                                                                                                                                                                          |
| **`DELU`**   | `[esd12=0.01] [esd13=0.01]`               | 'Rigid bond' restraint.                                                                                                                                                                                                                                                                                                                                                                           |
| **`ISOR`**   | `[esd=0.1] [esd_terminal=0.2]`            | Restrains ADP of given atom(s) to be approximately isotropic.                                                                                                                                                                                                                                                                                                                                     |
| **`SAME`**   | `N`                                       | Splits selected atoms into `N` groups and applies SAME restraint. Olex2 manages atom order, but mixing with rigid groups might cause issues.                                                                                                                                                                                                                                                        |
| **`showp`**  | `[part_numbers]`                          | Shows only specified parts (space separated). `showp 0 1` shows parts 0 and 1. `showp` alone shows all parts.                                                                                                                                                                                                                                                                                        |
| **`split`**  | `[-r={eadp, isor, simu}]` (for selected atoms) | Splits selected atom(s) along longest ADP axis into two groups, links occupancy via free variable. `-r`: adds EADP, ISOR, or SIMU for generated atoms.                                                                                                                                                                                                                                          |
| **`AFIX`**   | `shelx_afix_number{mn} [-n]`              | Applies SHELX AFIX instruction. Auto-detects rings for some AFIX types (e.g., 106, 79) or expects selection. Special AFIX 0, 1, 2 for removing AFIX, fixing all params, or fixing coords only. `-n`: consider N-atoms in rings.                                                                                                                                                                      |
| **`part`**   | `[part=new_part] [atoms] [-lo] [-p=1]`    | Changes part number for atoms. `-lo`: links occupancies (via +/- variable or SUMP depending on `-p`). `-p`: number of parts to create. If `-p=1`, `-lo` is ignored.                                                                                                                                                                                                                                    |
| **`fvar`**   | `[value] [atoms]`                         | Links occupancies of atoms via free variable. No atoms: prints current fvars. No value, 2 atoms: links their occu with new fvar. Value 0: refines occu freely. Other value: sets occu to that value.                                                                                                                                                                                                  |
| **`sump`**   | `[val=1] [esd=0.01]` (for selected atoms) | Creates a linear equation for occupancies of selected atoms, summing to `val`.                                                                                                                                                                                                                                                                                                                            |
| **`mode split`**| `[-r={eadp, isor, simu}]`               | Splits subsequently clicked atoms into parts. Shift+click drags atom. Original and generated atoms go to different parts. `-r`: adds restraints/constraints.                                                                                                                                                                                                                                          |

### HKL file Operations

| Command        | Syntax                               | Description                                                                                                                                                                                                                                                             |
|----------------|--------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **`hklstat`**  |                                      | Prints detailed information about reflections used in refinement.                                                                                                                                                                                                         |
| **`omit`**     | `h k l`                              | Inserts `OMIT h k l` instruction (for Shelx).                                                                                                                                                                                                                             |

| **`appendhkl`**| `-h=h1;h2.. -k=k1;k2.. -l=l1;l2..`     | Opposite of `excludehkl`; appends specified reflections.                                                                                                                                                                                                                |
*Note: For advanced HKL processing (e.g., twinning), Python scripts like `hklf5.py` can be used (`@py -l` to load).*

#import "PZViewController.h"

@interface PZViewController ()

@end

@implementation PZViewController

- (void)viewDidLoad
{
    [super viewDidLoad];
	// Do any additional setup after loading the view, typically from a nib.

    _imageView1 = [[UIImageView alloc] initWithImage:[UIImage imageNamed:@"sun.png"]];
    [self.view addSubview:_imageView1];
    _imageView1.contentMode = UIViewContentModeBottomLeft;

    _imageView2 = [[UIImageView alloc] initWithImage:[UIImage imageNamed:@"mountains.png"]];
    [self.view addSubview:_imageView2];
    _imageView2.contentMode = UIViewContentModeBottomLeft;

    _imageView3 = [[UIImageView alloc] initWithImage:[UIImage imageNamed:@"grass.png"]];
    [self.view addSubview:_imageView3];
    _imageView3.contentMode = UIViewContentModeBottomLeft;

    _label = [[UILabel alloc] initWithFrame:CGRectMake(0, 0, 320, 100)];
    _label.text = @"Parallax";
    _label.textAlignment = NSTextAlignmentCenter;
    _label.font = [UIFont systemFontOfSize:40];
    _label.textColor = [UIColor whiteColor];
    _label.backgroundColor = [UIColor clearColor];
    [self.view addSubview:_label];

    _imageView1.frame = CGRectMake(0, 0, _imageView1.image.size.width, _imageView1.image.size.height);
    _imageView2.frame = CGRectMake(0, 0, _imageView2.image.size.width, _imageView2.image.size.height);
    _imageView3.frame = CGRectMake(0, 0, _imageView3.image.size.width, _imageView3.image.size.height);


    // Set vertical effect
    UIInterpolatingMotionEffect *verticalMotionEffect =
    [[UIInterpolatingMotionEffect alloc]
     initWithKeyPath:@"center.y"
     type:UIInterpolatingMotionEffectTypeTiltAlongVerticalAxis];
    verticalMotionEffect.minimumRelativeValue = @(-20);
    verticalMotionEffect.maximumRelativeValue = @(20);

    // Set horizontal effect
    UIInterpolatingMotionEffect *horizontalMotionEffect =
    [[UIInterpolatingMotionEffect alloc]
     initWithKeyPath:@"center.x"
     type:UIInterpolatingMotionEffectTypeTiltAlongHorizontalAxis];
    horizontalMotionEffect.minimumRelativeValue = @(-20);
    horizontalMotionEffect.maximumRelativeValue = @(20);

    // Create group to combine both
    UIMotionEffectGroup *group = [UIMotionEffectGroup new];
    group.motionEffects = @[horizontalMotionEffect, verticalMotionEffect];

    // Add both effects to your view
    [_label addMotionEffect:group];

    // Set vertical effect
    UIInterpolatingMotionEffect *verticalMotionEffect1 =
    [[UIInterpolatingMotionEffect alloc]
     initWithKeyPath:@"center.y"
     type:UIInterpolatingMotionEffectTypeTiltAlongVerticalAxis];
    verticalMotionEffect1.minimumRelativeValue = @(-10);
    verticalMotionEffect1.maximumRelativeValue = @(10);

    // Set horizontal effect
    UIInterpolatingMotionEffect *horizontalMotionEffect1 =
    [[UIInterpolatingMotionEffect alloc]
     initWithKeyPath:@"center.x"
     type:UIInterpolatingMotionEffectTypeTiltAlongHorizontalAxis];
    horizontalMotionEffect1.minimumRelativeValue = @(-10);
    horizontalMotionEffect1.maximumRelativeValue = @(10);

    // Create group to combine both
    UIMotionEffectGroup *group1 = [UIMotionEffectGroup new];
    group1.motionEffects = @[horizontalMotionEffect1, verticalMotionEffect1];

    // Add both effects to your view
    [_imageView1 addMotionEffect:group1];

    // Set vertical effect
    UIInterpolatingMotionEffect *verticalMotionEffect2 =
    [[UIInterpolatingMotionEffect alloc]
     initWithKeyPath:@"center.y"
     type:UIInterpolatingMotionEffectTypeTiltAlongVerticalAxis];
    verticalMotionEffect2.minimumRelativeValue = @(-5);
    verticalMotionEffect2.maximumRelativeValue = @(5);

    // Set horizontal effect
    UIInterpolatingMotionEffect *horizontalMotionEffect2 =
    [[UIInterpolatingMotionEffect alloc]
     initWithKeyPath:@"center.x"
     type:UIInterpolatingMotionEffectTypeTiltAlongHorizontalAxis];
    horizontalMotionEffect2.minimumRelativeValue = @(-5);
    horizontalMotionEffect2.maximumRelativeValue = @(5);

    // Create group to combine both
    UIMotionEffectGroup *group2 = [UIMotionEffectGroup new];
    group2.motionEffects = @[horizontalMotionEffect2, verticalMotionEffect2];

    // Add both effects to your view
    [_imageView2 addMotionEffect:group2];
}

-(void)viewDidLayoutSubviews
{
    [super viewDidLayoutSubviews];
    _imageView1.frame = CGRectMake(0, 0, _imageView1.image.size.width, _imageView1.image.size.height);
    _imageView2.frame = CGRectMake(0, 0, _imageView2.image.size.width, _imageView2.image.size.height);
    _imageView3.frame = CGRectMake(0, 0, _imageView3.image.size.width, _imageView3.image.size.height);
    _label.center = self.view.center;
}

- (void)didReceiveMemoryWarning
{
    [super didReceiveMemoryWarning];
    // Dispose of any resources that can be recreated.
}

@end
